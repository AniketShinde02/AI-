# Nexus Voice Architecture (Stable v1)

## Overview
Nexus features a resilient, dual-engine voice AI pipeline that provides state-of-the-art natively multimodal interactions, while maintaining a robust text-to-speech fallback to guarantee high availability.

## Frontend Pipeline (`useNexusVoice.ts`)
The React frontend manages audio state via the `useNexusVoice` hook. 
- **Audio Capture:** Uses the MediaRecorder API to capture 16kHz PCM audio. 
- **VAD (Voice Activity Detection):** Detects user speech bounds. When speech stops, it emits a `vad_stop` WebSocket payload to notify the server to begin inference.
- **WebSocket Transport:** Maintains a persistent bidirectional WebSocket connection with the FastAPI backend, streaming base64 encoded audio chunks up, and receiving base64 encoded TTS audio chunks down.
- **Audio Playback:** Queues and plays incoming audio chunks sequentially using the Web Audio API to ensure gapless playback.

## Backend Pipeline

### 1. `ws_main.py` & `api/websocket_routes.py`
The FastAPI WebSocket endpoint that acts as the ingress for the frontend.
- It receives settings, user text, and raw audio chunks.
- It maintains a global map of active connections and maps them to isolated `VoiceSession` objects.

### 2. `VoiceSession` (`core/voice_session.py`)
The orchestrator for a single user's interaction loop.
- Uses a unified interface to process text and audio input.
- Spawns background workers (`tts_worker` and `metrics_worker`) to asynchronously process outputs.
- Contains the **Fallback Router** logic.

### 3. `GeminiLiveManager` (`core/gemini_live_manager.py`)
The primary multimodal interface connecting to Google's Native Audio Dialog API (`gemini-2.5-flash-native-audio-latest`).
- Opens a persistent server-to-server WebSocket (`google-genai` SDK).
- Continuously streams user mic PCM data.
- Handles the Half-Duplex turn constraint by monitoring `vad_stop` signals.
- Streams live TTS chunks back to `VoiceSession` as they are generated by the model.
- Automatically handles SDK session closures and cleanly re-establishes connections via a `while True` receive loop.

### 4. Fallback Chain
If the Gemini Live API drops, errors out, or is explicitly disabled, the `VoiceSession` immediately drops to a standard transcription-LLM-TTS pipeline:
1. **Groq STT:** Whispers user audio to text.
2. **Groq LLM:** Fast inference on LLaMA 3.
3. **EdgeTTS:** Synthesizes the response to audio chunks.

## Critical System Warning
The WebSocket lifecycle in `gemini_live_manager.py` and `websocket_routes.py` is extremely fragile. The session uses precise asynchronous task cancellation and queue management to guarantee non-blocking audio streaming.
**Do not modify without full E2E testing.**
