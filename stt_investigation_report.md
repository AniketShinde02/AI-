# STT & Intent Routing Investigation Report

This report documents our findings regarding the Speech-to-Text (STT) pipeline, Marathi language handling, and intent classification routing for the Nexus V1 platform.

---

## 1. Active STT Provider in Gemini Live Mode

*   **Active Provider:** **Groq STT (Whisper-large-v3)** is the primary active STT provider, with **Gemini 2.5 Flash** (via `generate_content` multipart byte payload) as the fallback.
*   **Audio Flow:** In `/ws/gemini-live` mode, raw audio bytes are captured locally, processed through local Silero VAD, and packaged as a WAV file. The file is sent to Groq STT (or Gemini 2.5 Flash as a fallback).
*   **Gemini Live Input:** Gemini Live is **not** processing raw audio streams directly for reasoning. Instead, the local STT output is converted to text and routed as client content:
    ```python
    await self.gemini_manager.send_text(transcript, turn_complete=True)
    ```

---

## 2. Transcript Processing (Gemini Live vs. Groq)

*   **Double Processing:** Yes, both pipelines process the same transcript under the hood. 
*   **Routing Logic:**
    1. The transcript is generated by the local STT layer (Groq Whisper or Gemini fallback).
    2. The transcript is sent to `run_llm_and_tts()`.
    3. `action_router.route_intent(transcript)` intercepts the text first to classify it as an `ACTION` (system tool execution) or `CHAT` (conversation).
    4. If classified as `CHAT` and the engine is `gemini_live`, it delegates to the Gemini Live session via `send_text()`. If the engine is local `nexus`, it routes to `GroqChat` via the standard LLM completion.

---

## 3. Marathi Speech Recognition Accuracy

*   **Transcription Resiliency:** Whisper-large-v3 supports Marathi, but accuracy drops significantly under the following conditions:
    *   **Code-switching:** Mixing Marathi and English words (e.g. Marathi instructions referencing English app names like VS Code).
    *   **Dialects and Accentuation:** Standard Whisper-large-v3 model parameters can misinterpret regional Marathi pronunciation.
*   **Gemini Fallback:** Gemini 2.5 Flash performs better with contextual understanding of Marathi but introduces increased latency.

---

## 4. Confidence Score Availability

*   **API Limits:** The Groq STT/Whisper API does not return a standardized confidence score (e.g. `0.0` to `1.0`).
*   **Confidence Proxies:** The system utilizes segment-level log probability (`avg_logprob`) and no-speech probability (`no_speech_prob`) returned by `verbose_json` as proxies:
    ```python
    if avg_no_speech > 0.6 or avg_logprob < -1.0:
        # Rejected as hallucination
    ```

---

## 5. Language Detection Path

*   **Implicit Detection:** Language detection is handled implicitly by Groq Whisper-large-v3 during audio transcribing.
*   **Prompt Guiding:** To assist detection and prevent phonetic mismatching, the system passes a multilingual prompt context:
    ```python
    multilingual_prompt = (
        "English, Hindi, and Marathi conversation. Technical AI assistant context. "
        "User may use Hinglish or Marathi speech. "
        "Examples: 'kaise ho?', 'kya chal raha hai?', 'काय चाललंय?', 'Show me code.', 'explain this'."
    )
    ```
*   There is no active software gate/sensor in Python to inspect or branch based on the detected language code before processing the text.

---

## 6. Why Corrupted Marathi Text Reaches Tool Execution

*   **Issue 1: Missing Action Router Guidelines:** The `ActionRouter` (running on `llama-3.1-8b-instant` with temperature `0.0`) has system prompt guidelines specifically tuned for *Hinglish* but has **zero rules/examples for Marathi structure**.
*   **Issue 2: Phonetic Sound-Alike Mismatching:** If a user speaks Marathi and the STT layer produces a phonetically corrupted or Devanagari transcription, the `ActionRouter` struggles to identify it as conversational chat. If any part of the utterance phonetically resembles tool keywords (e.g., "code", "close", "run"), it assumes it is a command intent.
*   **Issue 3: Strict JSON Forcing:** The router is forced to output JSON schema attributes (`"tool"`, `"target"`). In the absence of Marathi conversational context, the model fails to return `"null"` and instead forces a mapping to a system tool name (e.g. `pc_open_app`), sending the corrupted Marathi string directly to tool execution.
